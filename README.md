# Steam Key Google Sheet Batch Query

This tool was made to facilitate the monitoring of activation status of Steam Keys being sent out to content creators via a mail merge tool since Steam's web gui only does one at a time. It assumes the keys are in a Google Spreadsheet that you own, and that there is some column in that sheet indicating whether the key has been sent or not.

Credit to https://gist.github.com/Jimbly/9f6c6a0d9414310347f2803902ac7bb7 for the basis of this tool. If you just want to batch query a set of Steam Keys outside of a google sheet I recommend using Jimbly's solution instead, as it doesn't require the complexity of setting up or making requests to Google APIs.

**HEY YOU, INTERNET STRANGER: This tool makes web requests leveraging auth from accounts you probably care about. To make matters worse, this was also mostly generated by AI since JS isn't my bread and butter. ## 🛠 Prerequisites

* **Node.js**: [Download and install](https://nodejs.org/) (v14 or higher recommended).
* **Steamworks Partner Access**: You must have an account with permissions to query CD keys.
* **Google Cloud Project**: To access the Google Sheets API. [Google Cloud Project Setup Instructions](https://developers.google.com/workspace/guides/create-project)

## ✅ Setup Instructions

### 1. Project Installation

Clone or download this repository to your local machine.


### 2. Google Cloud & Sheets API Configuration

The script uses a Service Account to interact with your spreadsheet.

1. **Enable the API**: Go to the [Google Cloud Console](https://console.cloud.google.com/), create a project if you do not have one yet, and enable the **Google Sheets API**.
2. **Create or use an existing Service Account**:
* Navigate to **IAM & Admin > Service Accounts**.
* Create a new account (no specific project roles required).

3. **Generate a JSON Key**:
* Click on the created Service Account > **Keys** tab > **Add Key** > **Create New Key** > **JSON**.
* Save this file in your project folder as `google_credentials.json`.

4. **Share the Spreadsheet**:
* Open your Google Sheet.
* Click **Share** and add the `client_email` address found inside your `google_credentials.json` as an **Editor**.


### 3. Configure `config.json`

Fill out `config.json`:

```json
{
  "spreadsheetId": "YOUR_SPREADSHEET_ID",
  "sheetName": "Sheet1",
  "googleApiAuthFile": "google_credentials.json",
  "keyColumn": "A",
  "resultColumn": "B",
  "filterColumn": "C",
  "filterValues": ["EMAIL_OPENED", "EMAIL_CLICKED"],
  "authCookies": "sessionid=...; steamLoginSecure=...;"
}

```

* **`spreadsheetId`**: Found in the URL of your sheet: `docs.google.com/spreadsheets/d/SPREADSHEET_ID_HERE/edit`.
* **`keyColumn`**: Letter id of column containing Steam Keys.
* **`resultColumn`**: Letter id of column where key activation status will be written to.
* **`filterColumn`**: Letter id of column that will indicate whether the Steam Key should be queried or not.
* **`filterValues`**: List of values that indicate the key should be queried. **Keys without one of these values will not be queried.**
* **`authCookies`**:
1. Log in to the [Steam Partner Portal](https://partner.steamgames.com/).
2. Press `F12` > **Storage** (Firefox) or **Application** (Chrome) tab > **Cookies** > **https://partner.steamgames.com**.
3. Copy the values for `sessionid` and `steamLoginSecure`.

## 🏃 Running the Script

You can use the provided runners:

* **Windows**: Double-click `run_query.bat`
* **Mac/Linux**: Run `./run_query.sh`

Alternatively, run the script directly via Node:

```bash
npm install
node index.js
```

## ⚠️ Troubleshooting

* **`Auth failed`**: Steam sessions expire. If you see this, refresh your login on the Steam Partner site and update the `sessionId` and `authCookies` in `config.json`.
* **`Could not load default credentials`**: Make sure the client email found in `google_credentials.json` is added as an editor to the spreadsheet.
